# Агент Публикатор

Автоматическая публикация контента из Notion в Telegram, VK и другие каналы.

## Структура проекта

```
publisher-agent/
├── src/
│   ├── channels/          # Модули публикации в каналы
│   │   ├── telegram.py    # Telegram канал
│   │   ├── vk.py          # VK группа
│   │   └── html_generator.py  # Генерация HTML для Тильды
│   ├── notion_client.py   # Работа с Notion API
│   └── main.py           # Главный файл (точка входа)
├── config/
│   └── settings.py       # Конфигурация и переменные окружения
├── tests/                # Тесты
├── requirements.txt      # Зависимости
├── .env.example          # Пример переменных окружения
└── README.md            # Этот файл
```

## Быстрый старт

### 1. Установка зависимостей

```bash
pip install -r requirements.txt
```

### 2. Настройка переменных окружения

Скопируй `.env.example` в `.env` и заполни своими данными:

```bash
cp .env.example .env
```

### 3. Получение API ключей

#### Notion
1. Перейди на https://www.notion.so/my-integrations
2. Нажми "New integration"
3. Скопируй "Internal Integration Token"
4. Подключи интеграцию к своей базе данных (Share → Invite → Выбери интеграцию)

#### Telegram
1. Напиши @BotFather в Telegram
2. Создай бота командой `/newbot`
3. Скопируй токен бота
4. Добавь бота администратором в канал
5. Узнай ID канала (можно через @userinfobot или @getidsbot)

#### VK
1. Перейди на https://vk.com/dev
2. Создай Standalone-приложение
3. Получи Access Token с правами `wall`, `photos`, `groups`
4. Узнай ID группы (число в адресе группы)

### 4. Запуск

```bash
python src/main.py
```

## Структура базы Notion

Создай базу данных с полями:

| Поле | Тип | Описание |
|------|-----|----------|
| **Название** | Title | Заголовок поста |
| **Статус** | Select | Черновик / Готово к публикации / Опубликовано |
| **Каналы** | Multi-select | Telegram, VK, Email, HTML |
| **Дата публикации** | Date | Когда опубликовать |

Контент поста пишется в теле страницы (блоки).

## Логика работы

1. Агент проверяет базу Notion каждые 5 минут
2. Находит страницы со статусом "Готово к публикации"
3. Забирает контент (текст, заголовки, списки, картинки)
4. Публикует в выбранные каналы
5. Меняет статус на "Опубликовано"
